# 每日信号与复盘流程

## 每日会发生什么

当前系统有两个自动任务：

1. 每日交易信号
2. 次日早上复盘

两者都会通过新的 Telegram bot 推送，不再和旧的青铜C内容混在一起。

## 每日交易信号

时间：

```text
每天 20:15 America/New_York
```

作用：

- 读取 LBank 日线数据
- 计算当前策略目标仓位
- 推送是否持仓、持有哪些币、每个币目标权重
- 如果没有机会，推送空仓

信号示例：

```text
LBank 做多策略信号
日期: 2026-07-05
模拟账户: 2000.00 USDT
目标仓位:
- 空仓
资金费率: 暂不可用，纸面账户按 0 计算
方向: 只做多；做空模块后续单独开发
```

## 次日复盘

时间：

```text
每天 08:00 America/New_York
```

作用：

- 更新 30 日纸面账户结果
- 复盘前一日策略表现
- 输出当前权益、盈亏、最大回撤、交易数和当前持仓

复盘示例：

```text
LBank 前一日复盘 / 30日纸面账户
区间: 2026-06-06 至 2026-07-05
初始: 2000.00 USDT
当前: 2000.00 USDT
盈亏: +0.00 USDT (+0.00%)
最大回撤: 0.00%
30日交易数: 0
当前持仓: 空仓
```

## 数据异常处理

如果 LBank 某个交易对临时请求失败，脚本会优先使用本地缓存，避免整条 Telegram 推送失败。

如果资金费率接口不可用，报告会写：

```text
资金费率: 暂不可用，纸面账户按 0 计算
```

## 旧青铜C推送

旧青铜C每日推送目前没有确认停掉，因为它不是由当前 Codex automation 管理的。新的策略推送已经使用新 Telegram bot，避免和旧内容混在一起。

如果要停旧推送，需要到旧系统里查：

- OpenClaw 定时任务
- 服务器 crontab
- Telegram bot 后台
- 其他自动化服务
